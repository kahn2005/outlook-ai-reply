# 📋 PRD — 제품 요구사항 문서

**제품명**: Outlook AI 답변 자동 초안 생성기
**작성일**: 2026-03-27
**버전**: v1.0

---

## 1. 제품 개요 (Overview)

Outlook(Microsoft 365)에서 수신된 이메일을 AI가 자동으로 분석하여, 수신 언어(일본어/한국어)에 맞는 답변 초안을 생성하고 Outlook 답장 창에 자동으로 입력해주는 생산성 도구.

---

## 2. 문제 정의 (Problem Statement)

| 항목 | 내용 |
|------|------|
| 사용자 | Microsoft 365 Outlook 업무 사용자 (貞善님) |
| 언어 환경 | 한국어 ↔ 일본어 혼용 이메일 처리 |
| 이메일 규모 | 하루 10~30통 |
| 핵심 불편함 | 수신 메일 내용을 파악하고 적절한 답변을 작성하는 반복 작업 |
| 현재 해결 방법 | 없음 (수동 작성) |

**고통 포인트:**
- 매번 메일을 읽고 핵심 요청사항을 파악하는 데 시간 소요
- 일본어 경어체/표현 선택의 어려움 및 심리적 부담
- 비슷한 유형의 답변을 반복해서 작성하는 비효율

---

## 3. 목표 (Goals)

- **주요 목표**: 이메일 답변 초안 작성 시간을 70% 이상 단축
- **부목표 1**: 언어(일본어/한국어) 자동 감지 및 동일 언어로 답변 생성
- **부목표 2**: 수신 메일의 핵심 내용을 정확히 반영한 답변 초안 제공
- **부목표 3**: 사용자가 최소한의 수정으로 전송 가능한 품질의 초안 생성

---

## 4. 핵심 기능 요구사항 (Functional Requirements)

### F1. 이메일 자동 감지 및 분석
- 새 이메일 수신 시 자동으로 내용 분석 시작
- 발신자, 제목, 본문을 분석하여 요청 사항 파악
- 이메일 유형 분류 (문의, 요청, 확인, 감사 등)

### F2. 언어 자동 감지
- 수신 이메일의 언어(일본어/한국어) 자동 판별
- 동일 언어로 답변 초안 생성

### F3. AI 답변 초안 생성
- Claude API를 활용한 자연스러운 답변 초안 생성
- 핵심 요청사항을 반영한 구체적 내용 포함
- 비즈니스 적절한 존댓말/경어체 자동 적용
- 이메일 기본 형식 준수 (인사말 → 본문 → 맺음말)

### F4. Outlook 자동 입력
- 생성된 초안을 Outlook 답장 작성 영역에 자동 삽입
- 사용자는 내용 확인 후 수정하여 전송

---

## 5. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|---------|
| 응답 속도 | 초안 생성 5초 이내 |
| 보안 | 이메일 내용은 API 처리 후 저장하지 않음 |
| 호환성 | Microsoft 365 Outlook (Windows) |
| 언어 지원 | 한국어, 일본어 |

---

## 6. 기술 스택 (Tech Stack)

| 구성 요소 | 기술 |
|-----------|------|
| AI 엔진 | Claude API (Anthropic) |
| Outlook 연동 | Microsoft Graph API 또는 Outlook Add-in (Office.js) |
| 백엔드 (선택) | Python / Node.js |
| 배포 방식 | Outlook 추가 기능(Add-in) 또는 로컬 스크립트 |

---

## 7. 사용자 시나리오 (User Story)

> **상황**: 일본 거래처에서 미팅 일정 조율 메일이 도착함

1. 메일 수신 → 시스템 자동 감지
2. AI가 메일 내용 분석: "3월 30일 오후 미팅 가능 여부 문의"
3. 일본어 답변 초안 자동 생성:
   - 인사말, 가능 여부 확인 요청, 대안 일정 제시 포함
4. Outlook 답장 창에 초안 자동 입력
5. 사용자가 내용 확인 → 필요 시 수정 → 전송

---

## 8. MVP 범위 (Minimum Viable Product)

**포함:**
- Claude API 연동 답변 초안 생성
- 한국어/일본어 자동 감지
- Outlook Add-in으로 버튼 클릭 시 초안 생성 및 삽입

**미포함 (추후 고려):**
- 메일 수신 시 완전 자동 생성 (보안 정책 고려 필요)
- 사용자별 커스텀 톤/스타일 학습
- 과거 대화 맥락 반영

---

## 9. 성공 지표 (Success Metrics)

- 답변 작성 시간: 평균 **5분 → 1분 이내**
- 초안 수정 없이 전송 비율: **30% 이상**
- 사용자 만족도: **4/5점 이상**
